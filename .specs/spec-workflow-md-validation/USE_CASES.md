# Use Cases

## UC-1: Проверка покрытия при работе с фичей @feature5 @feature6 @feature7

**Актор:** Разработчик

**Предусловия:**
- dev-pomogator установлен
- specs-workflow extension активен
- Существует полная фича в .specs/

**Основной сценарий:**
1. Разработчик отправляет промпт в Cursor/Claude
2. Хук beforeSubmitPrompt/UserPromptSubmit срабатывает
3. Система находит .specs/ папку
4. Система сканирует поддиректории на полноту
5. Для каждой полной фичи:
   - Парсит MD файлы (FR, AC, UC) на наличие @featureN
   - Парсит .feature файл на наличие # @featureN
   - Сравнивает теги
6. Генерирует validation-report.md
7. Выводит предупреждения в stdout

**Результат:** Разработчик видит статус покрытия

## UC-2: Пропуск валидации для неполных фич @feature3 @feature4

**Актор:** Разработчик

**Предусловия:**
- Существует директория .specs/wip-feature/
- Директория содержит только часть файлов (< 13)

**Основной сценарий:**
1. Хук срабатывает
2. Система находит .specs/wip-feature/
3. Система проверяет наличие всех 13 файлов
4. Обнаруживает что файлов меньше 13
5. Пропускает эту директорию

**Результат:** Нет ложных предупреждений для черновиков

## UC-3: Отключение через конфиг @feature9

**Актор:** Разработчик

**Предусловия:**
- Файл .specs-validator.yaml содержит enabled: false

**Основной сценарий:**
1. Хук срабатывает
2. Система находит .specs-validator.yaml
3. Читает enabled: false
4. Завершает работу без валидации

**Результат:** Валидация полностью отключена

## UC-4: Тихий выход без .specs/ @feature8

**Актор:** Разработчик

**Предусловия:**
- Папка .specs/ не существует

**Основной сценарий:**
1. Хук срабатывает
2. Система проверяет наличие .specs/
3. Папка не найдена
4. Завершает работу без вывода

**Результат:** Никаких сообщений, exit code 0
