# User Stories — Auto-Capture Layer

## US-1: Автозахват коррекций @feature1

Как разработчик, использующий Claude Code / Cursor с dev-pomogator,
я хочу чтобы мои коррекции AI-агента (отклонения tool use, фразы "нет, делай так", "actually...") автоматически сохранялись в очередь сигналов,
чтобы при запуске `/suggest-rules` они учитывались как pre-classified candidates и не терялись между сессиями.

## US-2: Быстрый просмотр накопленных сигналов @feature2

Как разработчик,
я хочу быстро просмотреть накопленные сигналы через `/reflect` без полного анализа сессии,
чтобы понимать какие паттерны коррекций накопились и при необходимости отклонить ложные срабатывания.

## US-3: Автоматическая дедупликация @feature3

Как разработчик,
я хочу чтобы `/suggest-rules` автоматически дедуплицировал новые кандидаты из очереди с существующими rules в `.claude/rules/`,
чтобы не создавать дубликаты правил и не тратить время на review уже покрытых тем.

## US-4: Кроссплатформенный захват @feature4

Как разработчик, работающий и в Claude Code, и в Cursor,
я хочу чтобы auto-capture работал на обеих платформах (с учётом ограничений Cursor по hook events),
чтобы сигналы накапливались независимо от используемой IDE.

## US-5: Автоматическое предложение обработки @feature5

Как разработчик, я хочу получать уведомление когда накопилось достаточно сигналов, чтобы не забывать запускать /suggest-rules.

> Источник идеи: Claudeception auto-activation hook

## US-6: Подтверждения усиливают сигналы @feature6

Как разработчик, я хочу чтобы мои одобрения ("отлично", "exactly") усиливали уже захваченные сигналы, чтобы наиболее подтверждённые паттерны получали приоритет в /suggest-rules.

> Источник идеи: claude-reflect-system approval patterns (MEDIUM confidence)
