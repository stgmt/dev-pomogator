# Non-Functional Requirements (NFR)

## Performance

- Phase -0.5 не должна добавлять значительной задержки к выполнению suggest-rules. Извлечение данных выполняется целенаправленно из 3-4 HTML-секций (friction categories, CLAUDE.md suggestions, big wins/usage patterns, project areas), а не полным парсингом файла.
- Приоритет извлечения определяет порядок обработки: friction categories > CLAUDE.md suggestions > usage patterns > big wins. Если файл велик, допускается обработка только первых двух категорий.

## Security

- N/A. Phase -0.5 читает только локальный файл из домашней директории пользователя (`~/.claude/usage-data/report.html`). Никакие данные не передаются вовне. Файл генерируется самим Claude Code через команду `/insights`.

## Reliability

- Graceful degradation: любая ошибка чтения файла (отсутствие, повреждение, невалидный HTML, отсутствие ожидаемых CSS-классов) приводит к тихому пропуску Phase -0.5 без ошибок и без блокировки дальнейшего выполнения.
- На платформе Cursor Phase -0.5 пропускается полностью и молча, без какого-либо вывода.
- Отсутствие insights не влияет на качество результатов остальных фаз (память + сессия работают как раньше).

## Usability

- Unified mode display после Phase -0.5 ясно показывает статус всех трёх источников данных (память, insights, сессия) одной компактной строкой с иконками.
- Stale-маркеры (`⚠️`) видны как в unified mode display, так и в таблицах Phase 3, чтобы пользователь мог оценить актуальность данных.
- Совет запустить `/insights` выводится только при отсутствии файла, а не при stale-данных, чтобы не раздражать пользователя.
- Маркер источника `📊 insights` / `📊 insights ⚠️` в Phase 3 позволяет отличить insights-кандидатов от session-кандидатов (`📍`) и memory-кандидатов (`🧠`).
