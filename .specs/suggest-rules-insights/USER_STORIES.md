# User Stories

- Как разработчик, использующий Claude Code, я хочу чтобы suggest-rules использовал кросс-сессионные паттерны из /insights, чтобы предлагаемые правила основывались на долгосрочных данных использования, а не только на одной сессии. @feature1 @feature2 @feature3 @feature4 @feature5 @feature6 @feature7
- Как разработчик, я хочу чтобы suggest-rules корректно пропускал insights когда отчёт отсутствует или устарел, чтобы команда никогда не блокировалась из-за недоступности данных. @feature2 @feature3
- Как разработчик, использующий Cursor, я хочу чтобы suggest-rules работал точно так же как раньше без каких-либо упоминаний insights, чтобы платформо-специфичная функциональность не мешала. @feature10
- Как разработчик, я хочу видеть в финальной таблице Phase 3 источник каждого кандидата (сессия, память или insights), чтобы понимать откуда взялась каждая рекомендация. @feature8 @feature9
