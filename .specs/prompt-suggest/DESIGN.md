## Компоненты

### 1. prompt_suggest_core.ts — Config, State, LLM, Utils

Общие утилиты для обоих хуков:
- `PromptSuggestConfig` — loadConfig() с auto-detect API
- `SuggestionState` — CRUD для `~/.claude/prompt-suggestion.json` (atomic write)
- `callSuggestionLLM()` — вызов Haiku с узким типом (не зависит от auto-commit)
- `extractFirstUserMessage()` — JSONL парсер (15 строк)
- `redactSecrets()` — копия из auto-commit (6 строк)
- `log()` — stderr logger

### 2. prompt_suggest_prompt.md — Системный промпт

Адаптация Claude Code Prompt Suggestion Generator v2:
- User perspective, "just about to type that" test
- Few-shot examples (coding-специфичные)
- NEVER SUGGEST list
- Silence option
- 2-12 слов, match user's style/language
- Контекст: user request + final response

### 3. prompt_suggest_stop.ts — Stop Hook

```
stdin (JSON) → parse → check stop_hook_active
                              │
                    ┌─────────┴──────────┐
                    │                    │
              stop_hook_active=true   false
                    │                    │
              approve (empty)    loadConfig → check enabled/apiKey
                                         │
                              extractFirstUserMessage(transcript_path)
                                         │
                              combine with last_assistant_message
                                         │
                              redactSecrets → read prompt.md
                                         │
                              callSuggestionLLM
                                         │
                              ┌──────────┴──────────┐
                              │                     │
                          empty (silence)      non-empty
                              │                     │
                         approve (empty)    writeSuggestionState
                                                    │
                                           output: {"decision": "approve",
                                                    "systemMessage": "💡 {suggestion}"}
```

### 4. prompt_suggest_submit.ts — UserPromptSubmit Hook

```
stdin (JSON) → parse → check enabled
                              │
                    prompt.trim() !== "+"
                              │
                         passThrough
                              │
                    (prompt === "+")
                              │
                    readState → expired?
                              │
                    ┌─────────┴──────────┐
                    │                    │
              expired/missing        valid
                    │                    │
              clear+passThrough    output: {additionalContext: "..."}
                                                    │
                                              clearState
```

## Где лежит реализация

- Extension source: `extensions/prompt-suggest/`
- Installed tools: `.dev-pomogator/tools/prompt-suggest/` (generated by installer)
- State file: `~/.claude/prompt-suggestion.json`
- Manifest: `extensions/prompt-suggest/extension.json`

## TypeScript Interfaces

```typescript
interface PromptSuggestConfig {
  enabled: boolean;
  ttl: number;         // ms, default 600000
  llm: {
    baseUrl: string;   // auto-detected
    apiKey: string;
    model: string;     // default: haiku
  };
}

interface SuggestionState {
  suggestion: string;
  timestamp: number;   // Date.now()
  sessionId: string;
}

interface StopHookInput {
  session_id?: string;
  transcript_path?: string;
  cwd?: string;
  stop_hook_active?: boolean;
  last_assistant_message?: string;
}

interface SubmitHookInput {
  session_id?: string;
  prompt?: string;
  cwd?: string;
}
```

## BDD Test Infrastructure

**Classification:** TEST_DATA_NONE
**Evidence:** Хуки работают с Claude Code runtime (stdin/stdout JSON protocol). Нет БД, нет API мутаций, нет shared state кроме temp state file. Тесты stateless — каждый сценарий описывает isolated поведение hook-а.
**Verdict:** Hooks/fixtures не требуются. BDD сценарии документируют поведение, ручная верификация через Claude Code runtime.
