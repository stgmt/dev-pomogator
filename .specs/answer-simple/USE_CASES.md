# Use Cases

## UC-1: Агент молча предотвращает жаргонный ответ (happy path для US-1) -- @feature1

Контекст: агент только что закончил длинную работу (например, обновил extension.json + добавил FR + подключил audit category) и готовит summary-ответ пользователю.

- Шаг 1: Агент готовит черновик "Готово. Обновил extension.json. Добавил FR-12. Подключил VARIANT_COVERAGE."
- Шаг 2: Срабатывает always-apply rule answer-simple, шаг 3 самопроверки "поймёт ли юзер?" — находит проблемы: "FR-12" (внутренний код без расшифровки), "VARIANT_COVERAGE" (имя audit category непонятно), нет причинно-следственной структуры (просто 3 точки "сделал X")
- Шаг 3: Срабатывает шаг 5 "переписать". Агент формирует микроисторию: "После того как ты попросил добавить новое требование про X — я зафиксировал его в спеке (FR-12, новое функциональное требование про Y потому что Z) и подключил audit-проверку которая ловит когда требование не покрыто тестом. Сейчас extension.json актуален. Дальше — запусти audit-spec.ts чтобы убедиться что проверка прошла."
- Результат: пользователь получает понятный ответ с контекстом (зачем добавлен FR, что делает audit), без необъяснённого жаргона

## UC-2: Пользователь явно вызывает /answer-simple для аудита черновика (happy path для US-2) -- @feature2

Контекст: пользователь сам пишет драфт сообщения в Slack / комментария в PR / документации и хочет проверить читабельность для своих коллег.

- Шаг 1: Пользователь набирает `/answer-simple "После того как мы зарелизили v0.2.5 (см. Wave 14 в spec foo-bar), нужно мигрировать Adapter X на новую FR-8 семантику"`
- Шаг 2: Skill парсит, находит проблемы: "Wave 14" — внутренний код, "spec foo-bar" — внутренняя ссылка, "FR-8" — внутренний код, "Adapter X" — может быть OK если контекст PR
- Шаг 3: Skill возвращает: "Переформулировано: 'После последнего релиза нужно обновить X-адаптер на новую версию протокола обмена данными'", "Найдено проблем: 3 внутренних кода (Wave 14, spec foo-bar, FR-8) — заменить на смысл или удалить если контекст не требует"
- Результат: пользователь видит конкретные проблемы своего текста и улучшенную версию

## UC-3: Триггер инцидента — пользователь говорит "не понял" (edge case для US-1) -- @feature1

Контекст: агент только что задал жаргонный вопрос (например "Issue B — Wave 14 vs Wave 11 — Keep / Swap / Parallel?"). Пользователь отвечает "ниче не понял слишком сложно".

- Шаг 1: Always-apply rule answer-simple детектит триггер инцидента ("не понял" в последнем user message)
- Шаг 2: Активируется специальная ветка workflow: запрет на новый вопрос
- Шаг 3: Агент перечитывает копипаст пользователя за последние 2-3 turn, выполняет шаг 1 самопроверки "что я понял" честно (без додумывания)
- Шаг 4: Если из контекста ответ выводится → действовать молча, изложить результат как микроисторию; если действительно нужно уточнение → задать ОДНУ свободную фразу без multi-select
- Результат: пользователь не входит в цикл "не понял → переспрос → не понял"

## UC-4: Установка extension в target-проект через installer (happy path для US-3) -- @feature3

Контекст: maintainer dev-pomogator (другой пользователь) подключает свой проект к dev-pomogator. Команда установки запускается из target-проекта.

- Шаг 1: `npx github:<owner>/dev-pomogator --claude --plugins=answer-simple` (где `<owner>` — runtime-derived GitHub-owner репозитория dev-pomogator) для non-interactive установки только этого extension, либо `npx github:<owner>/dev-pomogator --claude --all` для bundled установки всех extensions
- Шаг 2: Installer читает `extensions/answer-simple/extension.json`, проверяет integrity (rule extension-manifest-integrity)
- Шаг 3: Installer копирует файлы по путям из manifest: `.claude/rules/answer-simple/clear-questions-to-user.md`, `.claude/skills/answer-simple/SKILL.md` — в target-проект
- Шаг 4: Installer записывает SHA-256 хеши в `~/.dev-pomogator/config.json` `managedFiles` (для будущих апдейтов через updater)
- Шаг 5: Installer добавляет answer-simple строку в `.gitignore` managed-блок target-проекта (per personal-pomogator FR-2)
- Результат: в target-проекте появилось always-apply правило и `/answer-simple` slash-команда, оба работают сразу без ручной настройки
