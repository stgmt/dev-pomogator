# User Stories

- Как разработчик, я хочу чтобы тесты не запускались на устаревшем билде, чтобы не тратить время на ложные результаты. @feature1
- Как разработчик, я хочу автоматическую пересборку перед тестами когда код изменился, чтобы не помнить про `npm run build` вручную. @feature2
- Как разработчик, я хочу чтобы guard работал для всех фреймворков (vitest, dotnet, pytest, Docker), чтобы защита была универсальной. @feature3
- Как разработчик, я хочу видеть в statusline/TUI фазу "Building..." перед "Running...", чтобы понимать что происходит. @feature4
- Как разработчик, я хочу возможность пропустить проверку staleness (например `--skip-build-check`), чтобы в экстренных случаях запустить тесты быстро. @feature5
