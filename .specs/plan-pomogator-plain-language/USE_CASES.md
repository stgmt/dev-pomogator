# Use Cases

## UC-1: Happy path — план с simple summary в шапке @feature1

Стандартный сценарий: AI выводит понимание в чат, пользователь подтверждает, AI пишет план.

- Шаг 1: AI исследует задачу
- Шаг 2: AI выводит `## 💬 Простыми словами` контент в чат как обычное сообщение
- Шаг 3: Пользователь отвечает "да" / "верно" / "ок"
- Шаг 4: AI пишет план-файл с `## 💬 Простыми словами` первой секцией перед `## 🎯 Context`
- Шаг 5: AI вызывает ExitPlanMode → plan-gate валидация Phase 1 проходит
- Результат: пользователь видит монстр-план в диалоге уже подтвердив понимание

## UC-2: Correction loop — пользователь поправляет @feature1 @feature3

Пользователь обнаруживает непонимание на этапе чат-сообщения, AI исправляет до написания плана.

- Шаг 1: AI выводит "Простыми словами" v1 в чат
- Шаг 2: Пользователь: "не так — X не Y"
- Шаг 3: AI выводит "Простыми словами" v2 с исправленной интерпретацией
- Шаг 4: Пользователь подтверждает
- Шаг 5: AI пишет план и вызывает ExitPlanMode
- Результат: монстр-план соответствует реальной задаче без потерянных усилий

## UC-3: Uncertainty A/B/C variants @feature4

AI явно показывает варианты своей интерпретации когда не уверен.

- Шаг 1: AI выводит "Простыми словами" с подсекцией "Правильно понял?" содержащей варианты A/B/C
- Шаг 2: Пользователь: "B"
- Шаг 3: AI обновляет понимание под B и выводит финальную версию
- Шаг 4: Пользователь подтверждает
- Шаг 5: AI пишет план и вызывает ExitPlanMode
- Результат: правильная интерпретация выбрана без длинной дискуссии

## UC-4: Backward compat breaking — старый план без секции @feature5 @feature6 @feature7 @feature8

Существующий план старого формата не проходит валидацию после bump до 2.0.0.

- Шаг 1: Пользователь редактирует план без секции "Простыми словами"
- Шаг 2: ExitPlanMode → plan-gate Phase 1 → error "Отсутствует секция: Простыми словами"
- Шаг 3: Hint содержит шаблон секции с тремя подсекциями для копипасты
- Шаг 4: Пользователь добавляет секцию вручную
- Шаг 5: Перезапуск ExitPlanMode → проходит
- Результат: миграция выполнена осознанно, breaking change принят

## UC-5: Edge case — пустая секция @feature2

Секция добавлена но не заполнена контентом — `validateHumanSummarySection` ловит.

- Шаг 1: AI/пользователь создаёт `## 💬 Простыми словами` без контента (только heading)
- Шаг 2: ExitPlanMode → plan-gate Phase 1
- Шаг 3: validateHumanSummarySection: error "Секция Простыми словами пуста"
- Шаг 4: Hint: "Добавь Сейчас / Как должно быть / Правильно понял? с непустыми строками"
- Шаг 5: Заполнение → re-run → passes
- Результат: пустая секция гарантированно не проходит, ревью всегда видит реальный контент

## UC-6: Edge case — секция в неправильном порядке @feature2

Секция есть но после Context — относительный order check ловит.

- Шаг 1: Плагин содержит `## 🎯 Context` первым и `## 💬 Простыми словами` после
- Шаг 2: ExitPlanMode → validateSections в `validate-plan.ts:74-100`
- Шаг 3: Order check `if (index < lastIndex)` срабатывает: Context.index < ПростымиСловами.index не выполнено → но первая итерация ставит lastIndex=Context.index, на второй итерации ПростымиСловами.index < lastIndex=Context.index → error "Секция Простыми словами находится не в требуемом порядке"
- Шаг 4: Hint: "Порядок: Простыми словами → Context → User Stories → ..."
- Шаг 5: Пользователь перемещает секцию в начало
- Результат: позиция первая гарантирована через REQUIRED_SECTIONS массив + относительный order check

## Edge cases not enforced (out of validator scope)

Следующие edge cases НЕ проверяются validator'ом — это поведенческие проблемы покрываемые правилом, не Phase 1 errors:

- **Multiple `## 💬 Простыми словами` sections**: validator находит первое вхождение через `findHeadingIndex`, повторные секции игнорируются. Дубликаты не ломают валидацию но создают confusion. Покрывается правилом "не дублируй секции".
- **Typo в heading** (`## 💬 Просто словами`): regex `/^##\s+(?:💬\s+)?Простыми словами\s*$/` строгий — typo приводит к "Отсутствует секция", что корректно (тот же error message что и при полностью отсутствующей секции).
- **Missing emoji** (`## Простыми словами` без 💬): regex делает emoji опциональным `(?:💬\s+)?`, поэтому валидно. Это intended behavior.
- **AI не вывел секцию в чат перед ExitPlanMode**: validator не читает transcript (см. RESEARCH.md Rejected Alternative C). Покрывается правилом plan-pomogator.md инструкцией "AI ОБЯЗАН выводить в чат перед ExitPlanMode".
