## Verification Process

### How CHKs are verified

1. Each CHK is linked to at least one BDD scenario or manual review via Traces To.
2. Verification Method: `BDD scenario` | `Manual review` (no unit tests; integration-first per `integration-tests-first.md`).
3. Status advances only when linked test passes.

### Status lifecycle

`Draft` → `In Progress` → `Verified` → `Blocked` (regression).

### Review cadence

- Phase 2 STOP: all CHKs in `Draft`.
- Phase 3 STOP: ≥50% of CHKs in `In Progress` после прохождения Red baseline.
- Implementation end: 100% `Verified` или `Blocked` с issue link.
