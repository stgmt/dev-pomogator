# User Stories

- Как разработчик, использующий dev-pomogator, я хочу чтобы LSP-серверы автоматически устанавливались и настраивались при установке расширения, чтобы Claude Code получил семантическую навигацию по коду без ручной конфигурации. @feature1
- Как разработчик на TypeScript/JavaScript, я хочу чтобы vtsls LSP-сервер подключался к Claude Code автоматически, чтобы goToDefinition и findReferences работали из коробки. @feature2
- Как разработчик на Python, я хочу чтобы Pyright LSP-сервер настроился автоматически, чтобы Claude Code видел типы и диагностики Python-кода. @feature3
- Как разработчик на C#, я хочу чтобы csharp-ls интегрировался с Claude Code, чтобы семантическая навигация работала для .NET проектов. @feature4
- Как пользователь dev-pomogator, я хочу чтобы расширение проверяло наличие рантаймов (Node.js, Python, .NET SDK) перед установкой и сообщало о недостающих зависимостях, чтобы не получить сломанную установку. @feature5
- Как пользователь dev-pomogator, я хочу чтобы в CLAUDE.md автоматически добавлялись инструкции по использованию LSP (предпочтение goToDefinition над grep), чтобы Claude Code использовал LSP по умолчанию. @feature6
- Как разработчик, я хочу чтобы расширение поддерживало Piebald-AI marketplace как основной источник LSP-плагинов с fallback на ручную настройку, чтобы установка работала даже при недоступности маркетплейса. @feature7
