# User Stories

- **US-1** @feature1 — Как разработчик, дорабатывающий план после Phase 2 deny от plan-gate, я хочу видеть в deny-сообщении ТОЛЬКО актуальные промпты текущей сессии Claude Code, чтобы понимать реальный запрос пользователя и не путаться в обрывках чужого контекста.

- **US-2** @feature2 @feature4 @feature5 — Как разработчик, использующий plan-pomogator в нескольких параллельных Claude Code сессиях (на одной машине, в разных проектах), я хочу чтобы prompt-кэш одной сессии был полностью изолирован от другой, чтобы один проект не «заражал» промпты другого через общий файл `default.json`. Изоляция должна работать на write-side (отказ от создания default.json при отсутствии session_id) и на read-side (defense filter task-notification entries в legacy файлах).

- **US-3** @feature3 — Как разработчик, запускающий background задачи через Bash tool, я хочу чтобы их `<task-notification>` сообщения НЕ попадали в кэш `.plan-prompts-*.json` и не показывались в deny-сообщении plan-gate, потому что это системные нотификации, а не реальный пользовательский ввод.

- **US-4** — Как мейнтейнер dev-pomogator, я хочу чтобы баг подобного класса (cross-session leak в hook кэше) был зафиксирован через спеку с FR/AC/BDD сценариями (FR-6, FR-7 — meta-requirements), чтобы регрессия ловилась автоматически при будущих изменениях `prompt-capture` или `plan-gate`.
