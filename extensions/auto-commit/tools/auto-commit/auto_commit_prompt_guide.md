# Auto-Commit LLM Prompt Guide

> Guidelines for generating commit messages with gitmoji and Jira integration.

---

## Commit Message Format (Full)

```
{emoji} {type}({scope}): {short description max 50 chars}

[JIRA-KEY](https://jira.example.com/browse/JIRA-KEY)

## Summary
{2-3 sentences describing what was done and why}

## New Files
| File | Description |
|------|-------------|
| `{filename}` | {what this file does} |

## Modified Files
| File | Description |
|------|-------------|
| `{filename}` | {what changed} |

## Deleted Files (if any)
| File | Reason |
|------|--------|
| `{filename}` | {why deleted} |

---
Auto-generated by Cursor Agent
```

---

## Smart Commit Format (Optional)

```
JIRA-KEY #comment short summary of changes
```

Smart Commit rules:
1. **One line only** (no line breaks)
2. **No markdown, no quotes, no bullet points**
3. **Starts with Jira key** then `#comment`
4. **English only**, concise summary

## Rules (Full Format)

1. **Gitmoji FIRST** - `‚ú® feat(scope): ...`
2. **Jira clickable** - `[PROJ-123](https://jira.example.com/browse/PROJ-123)`
3. **Language**: English
4. **First line length**: max 72 characters
5. **All files**: list ALL changed files with descriptions
6. **Markdown tables**: use for file lists
7. **Footer**: `Auto-generated by Cursor Agent`

## Checklist Before Generation

- [ ] Gitmoji in first line?
- [ ] Correct type (feat/fix/refactor...)?
- [ ] Scope specified?
- [ ] Jira link clickable (if configured)?
- [ ] Summary clear?
- [ ] ALL files listed?
- [ ] Footer present?
