# Reqnroll Cucumber Expression — Slash Guard

## Проблема

Reqnroll 2.x (и SpecFlow последних версий) **авто-детектит** тип паттерна в `[Given/When/Then(...)]`:

| Паттерн содержит | Детектится как | `/` ведёт себя как |
|------------------|----------------|---------------------|
| regex-метасимволы (`.*`, `\d`, `^$`, скобки и т.д.) | **regex** | literal `/` ✓ |
| только простой текст + `{int}`/`{string}` | **Cucumber Expression** | **оператор альтернативы** ❌ |

В Cucumber Expression `apple/banana` = "apple ИЛИ banana". Если `/` в начале (`/path`) → пустая левая ветка → Reqnroll падает с ошибкой:

```
Alternative may not be empty. If you did not mean to use an alternative you can use '\/' to escape the '/'.
```

## Ловушка для AI агента

Частый случай — путь к API:

```csharp
// ❌ ПЛОХО — нет regex-маркеров, /v1 → пустая левая ветка альтернативы
[When(@"я запрашиваю список моделей через /v1/models")]

// ❌ ПЛОХО — USD/RUB парсится как альтернатива "USD" или "RUB"
[Then(@"курс USD/RUB корректен")]
```

**Почему не заметно на ревью:** паттерн с `@"..."` выглядит как regex (C# verbatim string), но Reqnroll смотрит на СОДЕРЖИМОЕ строки, а не на синтаксис C#.

## Как определить тип паттерна

Reqnroll детектит **regex** если в строке есть хотя бы один из:

| Маркер | Пример |
|--------|--------|
| Группа `(...)` | `(.*)`, `(\d+)`, `([A-Z])` |
| Символьный класс `[...]` | `[0-9]`, `[A-Za-z]` |
| Shorthand `\d`/`\w`/`\s`/`\b` | `\d+`, `\w+` |
| Escape `\.`, `\+`, `\?`, `\*` | `\.`, `\+` |
| Wildcard `.*`, `.+`, `.?` | `.*` |
| Anchors `^` в начале, `$` в конце | `^text$` |

Если **НИ ОДНОГО** из этих маркеров нет → Reqnroll парсит как **Cucumber Expression**.

## Правильно

### Вариант A — force regex через `^$` anchors (рекомендуется)

```csharp
// ✅ ^$ делает паттерн regex → / теряет спец-значение
[When(@"^я запрашиваю список моделей через /v1/models$")]
[Then(@"^курс USD/RUB корректен$")]
```

Минимально-инвазивно, `^$` не меняют семантику матчинга (Reqnroll всё равно матчит полную строку).

### Вариант B — escape `/` как `\/` (честный CE)

```csharp
// ✅ \/ — корректный escape для Cucumber Expression
[When(@"я запрашиваю список моделей через \/v1\/models")]
[Then(@"курс USD\/RUB корректен")]
```

Если паттерн действительно короткий без групп — можно оставить CE с эскейпами.

### Вариант C — добавить regex-группу для параметров

Если в паттерне есть захватываемые значения — добавь `(.*)` и автоматически получишь regex-детекцию:

```csharp
// ✅ (.*) → regex → / literal
[When(@"я запрашиваю модель ""(.*)"" через /v1/models")]
```

## Неправильно

```csharp
// ❌ Полагаться на @"..." (verbatim) как на "regex mode"
[Given(@"модель /v1/foo недоступна")]

// ❌ Думать что CE не применяется к простому тексту
[Then(@"проверяем a/b/c")]

// ❌ Оставлять URL-путь в паттерне без anchors/escape
[When(@"POST /api/v1/orders")]
```

## Автоматическая проверка

Hook `reqnroll-ce-guard` блокирует Write/Edit `.cs` файлов с такими step definitions и показывает оба варианта фикса. Hook проверяет:

1. Файл `*.cs`
2. Атрибут `[Given/When/Then/And(...)]`
3. Паттерн без regex-маркеров (из таблицы выше)
4. В паттерне есть неэскейпленный `/`

Если все 4 условия — DENY с указанием строки и фикса.

## Чеклист перед записью step definition

- [ ] Если в паттерне есть `/` — добавлены `^$` ИЛИ `/` эскейплен как `\/` ИЛИ есть `(.*)`/`\d`/`[...]`
- [ ] Для API-путей (`/v1/...`, `GET /api/...`) — предпочтительно `^$` anchors
- [ ] Для валют (`USD/RUB`) — либо `^$`, либо `\/`
- [ ] Для формул (`a / b`) — `\/`, иначе ломается матч
