# Verify Against Real Artifact — фикстуры зеркалят реальный вывод producer'а

Когда код парсит/ингестит вывод **внешнего инструмента** (cucumber NDJSON, `git`, CLI, API-ответ, компилятор, MCP-сервер) — ДВЕ вещи обязательны ПЕРЕД тем как назвать фичу готовой:

1. **Фикстуры зеркалят РЕАЛЬНЫЙ вывод** — не выдумывай поля, которых настоящий инструмент не эмитит.
2. **Прогон против ОДНОГО реального артефакта** + сверка с независимым ground-truth — а не только smoke на рукодельных данных.

Smoke на hand-built данных **не доказывает ничего** — он проверяет твою модель данных против самой себя.

## Инцидент (2026-06-03, spec-generator v4 FR-32)

`get_coverage` / honesty-gate строился на `parseNdjson`. 359 юнит-тестов зелёные, smoke на рукодельных нодах «работал». Реальная проверка на настоящем `.last-test-run.ndjson` (по настоянию advisor) показала: **всё врёт** — все 79 сценариев `passed`, хотя прогон был `40 passed / 10 pending / 29 undefined`. Два бага, которые фикстуры маскировали:

- **Windows-пути**: cucumber на Windows пишет uri через `\`, граф ключует через `/` → результаты не применялись вообще (все `undefined`).
- **worst-of-steps**: статус сценария дефолтил `PASSED`, если не было `FAILED`-шага → `UNDEFINED`/`PENDING` схлопывались в `PASSED`.

Почему 359 тестов не поймали: фикстуры клали `testCaseFinished.testStepResult.status` — **поле, которого реальный cucumber-js НЕ шлёт**. Парсер брал статус из фейкового поля, минуя реальную worst-of-steps логику. Классический **fake-positive**: тест проверял выдуманный контракт, не реальный.

## Правило

### Фикстуры внешнего вывода
- [ ] Фикстура **захвачена из реального инструмента** (`tool --format=… > fixture`) ИЛИ точно повторяет его envelope-форму (сверить ключи против живого вывода).
- [ ] **Ни одного поля, которого producer не эмитит.** Сомневаешься — сгенери реальный вывод и сравни ключи.
- [ ] Cross-platform учтён: пути (`\` vs `/`), переводы строк (`\r\n`), локали, кодировки.

### Verification перед «готово» / перед DONE
- [ ] Прогнал фичу против **одного реального артефакта** (реальный NDJSON / реальный git-репо / реальный API-ответ), не только синтетика.
- [ ] Сверил результат с **независимым ground-truth** (summary-строка самого инструмента, ручной подсчёт, второй источник). Числа совпали **точь-в-точь**.
- [ ] НЕ помечал задачу DONE по smoke с happy-path данными.

## Триггеры
- Пишешь/правишь parser / ingester / adapter для вывода внешнего инструмента.
- Собираешься назвать data-pipeline фичу готовой или пометить задачу DONE.
- Юнит-тесты зелёные, но против реального артефакта прогон не делал.
- Видишь в фикстуре поле — спроси себя: «реальный инструмент это реально шлёт?».
