# Proactive Investigation

## Правило

Агент исследует СРАЗУ, не спрашивая разрешения. Каждое утверждение подкреплено evidence.

Исследование, диагностика, проверка логов, grep по коду — это 🟢 Autonomous (Low Risk). Разрешение не нужно.

## Антипаттерны (ЗАПРЕЩЕНО)

Фразы-стопоры — агент НЕ ДОЛЖЕН их использовать:

- "Посмотреть?" / "Проверить?" / "Нужно ли проверить?"
- "Хотите чтобы я проверил?" / "Могу проверить, если хотите"
- "Shall I investigate?" / "Want me to check?" / "Should I look into?"
- Любая формулировка `{действие}?` где действие = исследование/проверка/диагностика

## Правильно

- Увидел ошибку → grep по коду → прочитал логи → показал результат с evidence
- Не уверен в причине → сформулировал 2-3 гипотезы → проверил каждую → показал evidence
- Не можешь проверить → пометил `[UNVERIFIED]` → указал что нужно для верификации

## Evidence формат

Каждое утверждение о коде/API/поведении ДОЛЖНО иметь evidence:

| Тип | Пример |
|-----|--------|
| grep | `grep -r "storages" src/ → найдено в file.cs:142` |
| код | `Строка 142: await client.CreateAsync(payload)` |
| лог | `Error: "You cannot associate..."` |
| скриншот | `Вижу: [текст]. Ожидал: [текст]. Результат: CONFIRMED/DENIED` |
| API | `POST /endpoint → 400: {"message": "..."}` |
| git | `git blame file.cs:142 → commit abc123` |

## Неправильно

```
Это не связано с нашими изменениями AcceptProduct.mslx.
Нужно проверить raw payload в логах. Посмотреть?
```

## Правильно

```
Это не связано с AcceptProduct.mslx.
Evidence: grep -r "storages" AcceptProduct.mslx → 0 результатов.
Ошибка возникает в CreatePurchaseReceiveAsync (file.cs:142),
вызов: await _zohoClient.CreatePurchaseReceiveAsync(payload).
Payload содержит storages[] для item с item-level bin tracking.
[UNVERIFIED] Нужен raw payload из логов для подтверждения
какой именно item вызывает ошибку.
```

## Чеклист

- [ ] Не спрашивал разрешение на исследование — сделал сам
- [ ] Каждое утверждение имеет evidence (grep/код/лог/скриншот)
- [ ] При неуверенности — проверил гипотезы, показал результат
- [ ] Непроверяемое помечено `[UNVERIFIED]` с указанием что нужно
