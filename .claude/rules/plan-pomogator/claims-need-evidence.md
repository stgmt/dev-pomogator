# Claims Need Evidence — план не строится на фантазии без пруфов

## Правило (always-apply для планов)

Любой **внешний / технический факт** в плане (возможности инструмента или библиотеки, «X
поддерживает Y», «быстрее», «по умолчанию», «требует», «совместим», поведение API/CLI,
числовые характеристики) ОБЯЗАН нести **метку-пруф** рядом с утверждением — либо находиться
в секции `## 🔎 Источники / Пруфы`. Утверждение без пруфа = **фантазия**, и в план оно
попадать не должно.

Это распространяет дисциплину `proactive-investigation` (каждое утверждение с evidence) на
**план целиком**: enforce'ится валидатором плана, а не держится на том, что юзер пнёт.

## Формат меток

| Метка | Когда | Пример |
|-------|-------|--------|
| `[src:<url>]` | веб-факт (док, спека, репо) | `cucumber-runner поддерживает perTest [src:https://stryker-mutator.io/docs/stryker-js/cucumber-runner/]` |
| `[ref:<file:line>]` | факт из кода/файла этого репо | `канон пишется через message-формат [ref:cucumber.json:5]` |
| `[cmd:<вывод>]` | проверено запуском | `один прогон ~8с [cmd:time cucumber → real 0m8.1s]` |

Допустима и обязательная секция-сводка `## 🔎 Источники / Пруфы` в конце смысловой части
плана: каждый нетривиальный факт → его пруф. Если факт под меткой — повторять в секции не
нужно.

## Что НЕ требует пруфа

- Доменные требования (FR/AC — это «должно быть», не факты).
- Очевидное из контекста задачи / уже доказанное в этом же диалоге.
- Собственные намерения («сделаю X», «перепишу Y») — это план, не факт о внешнем мире.
- Гипотезы — но они ДОЛЖНЫ быть явно помечены как гипотеза, а не выданы за факт
  (напр. «ожидаю ≥45% — гипотеза, проверяется замером», не «будет ≥45%»).

## Enforcement (валидатор)

`tools/plan-pomogator/validate-plan.ts` Phase 4 (`validateEvidence`):
1. Нет секции `## 🔎 Источники / Пруфы` → warning «внешние факты не подкреплены».
2. Секция есть, но без единого пруфа (url / `[src:]` / `[ref:]` / `[cmd:]`) → warning.
3. Строка-буллет в Context / Implementation Plan с claim-глаголом
   (`supports/поддерживает/быстрее/faster/требует/requires/умеет/exists/default/совместим`)
   без метки-пруфа рядом → warning «похоже на непроверенный факт без пруфа».

Сейчас это **предупреждения** (Phase 4, не блокируют — чтобы не ломать легаси-планы).
Escalation до ошибки Phase 1 — после прогона на корпусе существующих планов.

## Антипаттерн (реальный, из сессии 2026-06-20)

```
План: «официальный @stryker-mutator/cucumber-runner с coverageAnalysis perTest снимает
медленность; buildCommand предкомпилирует step-defs» — без единой ссылки.
```
Юзер: «ты гуглил? а то много всяких фактов без пруфов». При проверке доков выяснилось:
`buildCommand` **с tsx обычно НЕ нужен** [src:https://stryker-mutator.io/docs/stryker-js/configuration/] —
то есть один из «фактов» был выдумкой. Правило существует, чтобы такие утверждения
ловились валидатором ДО апрува, а не пользователем после.

## Чеклист перед ExitPlanMode

- [ ] В плане есть секция `## 🔎 Источники / Пруфы`
- [ ] Каждый внешний/технический факт несёт `[src:]`/`[ref:]`/`[cmd:]` или есть в секции
- [ ] Гипотезы помечены как гипотезы, не выданы за факты
- [ ] `validate-plan.ts` не флагает unsourced-claims (Phase 4 чисто по evidence)
