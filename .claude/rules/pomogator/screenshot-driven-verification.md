# Screenshot-Driven Verification

## Правило

КАЖДЫЙ скриншот РЕАЛЬНО анализировать. Не говорить "уже обработал" без описания что видно.

## Формат анализа

После Read screenshot:
```
Вижу: [точный текст statusline/UI]
Ожидал: [что должно быть после fix]
Результат: CONFIRMED / DENIED — [почему]
```

## Антипаттерн

- "Уже обработал" без описания
- "Работает" без скриншота-пруфа
- Несколько коммитов без визуальной верификации

## Чеклист

- [ ] Скриншот сделан
- [ ] Описано что ВИДНО (точный текст)
- [ ] Сравнено с ОЖИДАНИЕМ
- [ ] Расхождение объяснено (если есть)
