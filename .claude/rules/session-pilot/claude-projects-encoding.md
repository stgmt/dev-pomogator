# Claude Projects Path Encoding — WSL/Windows Variants

Claude Code stores conversation JSONLs at `~/.claude/projects/<encoded>/<uuid>.jsonl`. **Encoding rules differ between OS contexts**, even for the same logical worktree path. session-pilot must handle ALL variants.

## The encoding rules

When Claude Code runs in CWD `<path>`, it derives the project directory name as follows:

| OS context | Input CWD | Output `<encoded>` |
|---|---|---|
| WSL/Linux native | `/mnt/d/repos/foo` | `-mnt-d-repos-foo` |
| Windows native | `D:\repos\foo` | `D--repos-foo` |
| Cursor IDE Win | `D:\Users\stigm\.cursor\worktrees\foo` | `D--Users-stigm--cursor-worktrees-foo` |
| Mirrored mode (theoretical) | `\\wsl$\Ubuntu\home\user\foo` | (unverified — needs `--diagnose-livecycle`) |

## The cross-OS gotcha

User opens VSCode/Cursor on Windows → CWD reports as `D:\repos\foo`. Launches `claude` CLI in integrated terminal → JSONL written to `D--repos-foo`.

Same user opens WSL terminal in `/mnt/d/repos/foo` → launches `claude` → JSONL written to `-mnt-d-repos-foo`.

**Both refer to the SAME worktree on disk**, but different `~/.claude/projects/<encoded>` directories.

## Implementation contract

`encode_path_for_claude(p: str) -> list[str]` MUST return **all plausible variants** for input path:

```python
def encode_path_for_claude(p: str) -> list[str]:
    variants = set()
    p = p.rstrip("/").rstrip("\\")
    # Linux-style
    variants.add(p.replace("/", "-").replace(":", "").replace("\\", "-"))
    # Windows-style
    variants.add(p.replace(":", "").replace("\\", "-").replace("/", "-"))
    # If path starts with /mnt/X/, also produce X-... encoding
    if p.startswith("/mnt/") and len(p) > 6:
        drive = p[5].upper()
        rest = p[6:]
        variants.add(f"{drive}-{rest.replace('/', '-').replace(':', '')}")
    # If path starts with X:/, also produce -mnt-x-... encoding
    if len(p) >= 3 and p[1] == ":" and p[2] in ("/", "\\"):
        drive = p[0].lower()
        rest = p[3:].replace("\\", "-").replace("/", "-")
        variants.add(f"-mnt-{drive}-{rest}")
    return [v.lstrip("-") for v in variants] + [v for v in variants if v.startswith("-")] + list(variants)
```

The function returns BOTH leading-dash and no-leading-dash variants because Claude's encoding has been observed both ways (especially for Cursor `worktrees/foo` patterns).

## Both base directories must be scanned

```python
CLAUDE_PROJECTS_DIRS = [
    Path.home() / ".claude" / "projects",                    # WSL/Linux
    Path("/mnt/c/Users") / WINUSER / ".claude" / "projects", # Windows mount
]
```

When dashboard runs in WSL, `~/.claude/projects` is the WSL home (e.g. `/home/stigm/.claude/projects`). But Claude Code on Windows native writes to `C:\Users\stigm\.claude\projects` which from WSL shows as `/mnt/c/Users/stigm/.claude/projects`.

User may have BOTH (Claude Code installed on Windows AND in WSL). Both must be scanned + matched against ALL encoding variants.

## Diagnostic CLI as escape valve

For unforeseen variants:

```bash
python3 server.py --diagnose-livecycle /mnt/d/repos/some-worktree
```

Output dumps:
- All encoding variants generated by `encode_path_for_claude()`
- All base dirs scanned + their existence
- Per-JSONL match: full path, mtime, age, byte size
- Verdict: 🟢 LIVE / ⚪ idle / ❌ no match

If verdict is ❌, the diagnostic also lists actual Claude project dir names found, so user can identify the missing variant.

## Adding a new variant

When user reports a missing match:
1. Run `--diagnose-livecycle <path>` on the failing path
2. Compare output's «Claude project dirs» list with «Encoding variants» list
3. Identify the actual encoded directory used
4. Reverse-engineer the encoding rule (e.g., «paths under `\\wsl$\Ubuntu\` map to `wsl--Ubuntu-...`»)
5. Add new branch to `encode_path_for_claude()`
6. Add regression test in `tests/test_encode_path.py` with the exact failing path as fixture
7. Re-run diagnostic → confirm match

## What NOT to do

- ❌ Hardcode encoding patterns in business logic. Always use `encode_path_for_claude()` helper.
- ❌ Trust user-input path as already-encoded. Always pass through encoder.
- ❌ Cache encoded results across paths — encoding has known edge cases that warrant fresh evaluation.
- ❌ Strip leading dash. Some Claude project dirs DO start with dash (Linux-style).
