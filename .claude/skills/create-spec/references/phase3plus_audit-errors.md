# Audit Category 1: ОШИБКИ (Errors)

**Что это:** Расхождения между спецификацией и реальным кодом — несуществующие методы/классы, уже существующие компоненты помеченные как "Need to add", неверные имена сервисов/клиентов.

## Checks

1. Прочитать DESIGN.md секции про компоненты, reuse plan, файлы реализации — проверить что указанные файлы/классы/методы СУЩЕСТВУЮТ в кодовой базе (через `Glob`, `Grep`, `Read`)
2. Прочитать FILE_CHANGES.md — файлы с `action=edit` реально существуют, файлы с `action=create` ещё НЕ существуют (pre-implementation only)
3. Проверить правильность имён клиентов/сервисов/API в DESIGN.md и FR.md
4. Найти пометки "Need to add" / "TODO: create" — проверить что эти компоненты действительно отсутствуют

## Remediation

Для каждого finding:

- Несуществующий метод/класс → исправить ссылку на правильное имя ИЛИ переместить в "Need to add" если действительно нужно создать
- "Need to add" для существующего компонента → удалить пометку, указать существующее имя
- Неверное имя сервиса → grep по реальной кодовой базе, заменить во всех файлах спеки
- FILE_CHANGES `action=edit` для несуществующего файла → исправить на `action=create` ИЛИ убрать запись

## Severity

ERROR — несуществующая ссылка / неверное имя.
WARNING — pre-existing файл помечен как `create` (но не критично пока не реализуется).
