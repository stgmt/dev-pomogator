## Wave 14 (gates+OpenRouter) перед Wave 11 — Keep / Swap / Parallel?

Чтобы дать осмысленную рекомендацию, мне нужно понимать что именно содержат Wave 11 и Wave 14 в твоём контексте — я не знаю что у тебя в roadmap. Но могу дать общий фреймворк решения.

### Когда **Keep** (Wave 14 → 11, как запланировано)
- Wave 14 не зависит от артефактов Wave 11
- Gates (видимо, валидация/блокировки) логически должны быть установлены ДО фичи из Wave 11, чтобы её сразу разрабатывать "по правилам"
- OpenRouter нужен как инфраструктура, на которую Wave 11 будет опираться
- Откладывать gates = копить технический долг, который потом дороже исправлять

### Когда **Swap** (Wave 11 → 14)
- Wave 11 разблокирует пользователей / закрывает горящий запрос
- Wave 14 требует данных/паттернов, которые появятся только после Wave 11
- Gates без реального использования = преждевременная оптимизация (не на чем калибровать пороги)
- OpenRouter без потребителей = инфра-задел без верификации

### Когда **Parallel**
- Команда позволяет (≥2 человека / независимые контексты у AI-агента)
- Нет shared файлов/модулей → нет merge conflict
- Каждая Wave имеет свои тесты и может быть смержена независимо
- Риск: review bandwidth удваивается

4. Есть ли блокирующая зависимость одной Wave от другой?
