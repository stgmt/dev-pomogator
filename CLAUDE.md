## Architecture

**Extension/plugin system** for Claude Code that installs team coding standards, workflows, and tools into projects.

- **Source of truth**: `extensions/*/tools/` for tools, `.claude/rules/` for rules
- **Extensions** (`extensions/`): self-contained plugins for Claude Code
- **`.dev-pomogator/tools/`**: generated by installer from `extensions/` — not committed, listed in `.gitignore`
- **Updater** (`src/updater/`): autonomous background update system with content hashing and user modification protection
- **Config**: `~/.dev-pomogator/config.json` — tracks installed extensions, managed files with SHA-256 hashes
- **Hooks**: extension hooks installed to **project** `.claude/settings.local.json` for target projects (personal, gitignored by Claude Code convention — see `.specs/personal-pomogator/` FR-2). Team-shared `.claude/settings.json` is never touched in target projects — preserves team hooks. Self-guard: dev-pomogator's own repo uses `.claude/settings.json` for dogfooding (not settings.local.json). Only SessionStart check-update → global `~/.claude/settings.json`.
- **Personal-pomogator**: installer writes a managed marker block to target `.gitignore` listing all managed paths (tools, rules, skills, commands, settings.local.json) so `git add .` can't accidentally leak dev-pomogator files. See `.specs/personal-pomogator/`.
- Key extensions: `plan-pomogator`, `specs-workflow`, `suggest-rules`, `test-statusline`, `tui-test-runner` (hybrid Python/Node.js TUI test runner), `bun-oom-guard` (auto-patches Bun runner to prevent OOM on Windows), `context-menu` (Claude Code in Windows right-click via Nilesoft Shell)
